# Frontend Compilers

Several compilers generate Calyx programs from other high-level languages.

Installing the Python-based frontends with flit *can* still work. It is also possible (and perhaps preferred?) to manage these dependencies with ``uv``, as with the other Python components of Calyx.

The fud2-managed environment (i.e. one generated by ``fud2 env init``) includes all frontends by default.

If using ``uv sync`` within the base Calyx directory, passing the ``--all-extras`` flag will install / expose the front ends within the created virtual environment.

- [Dahlia](./dahlia.md): Dahlia is an imperative, loop-based programming language.
- [Systolic Array Generator](./systolic-array.md): Generates systolic arrays using parameters.
- [TVM Relay](./tvm-relay.md): Relay is an IR for the TVM framework to replace old computation graph based IRs.
- [NTT Pipeline Generator](./ntt.md): Generates a pipeline for the number theoretic transform.
- [Queues](./queues.md): Generates FIFO, PIFOs, and PIFO trees.
- [MrXL](./mrxl.md): A simple example frontend developed in the [frontend tutorial](../tutorial/frontend-tut.md).
